## 콘솔 사용 가이드

**Data & Analytics > Data Lake Storage > 콘솔 사용 가이드**

Data Lake Storage 서비스 콘솔은 다음 기능을 제공합니다.

* 버킷의 조회와 관리
* 자격 증명의 관리와 조회
* 객체의 목록 및 상세 정보 조회

## 버킷 조회하기

### 버킷 목록 조회

생성한 버킷 목록은 Data Lake Storage 서비스 콘솔에서 확인 가능합니다.

* 이름 왼쪽의 체크박스를 사용하여 버킷을 선택할 수 있습니다.
* 목록 상단 우측의 **새로고침**을 클릭하여 목록을 갱신할 수 있습니다.

| 항목 | 설명 |
| --- | --- |
| 이름 | 버킷의 이름 |
| 객체 개수 | 버킷에 저장된 객체의 수 |
| 크기 | 버킷에 저장된 객체의 총 크기 |

### 버킷 검색

버킷 목록 상단 좌측의 입력 영역과 돋보기 버튼을 사용하여 입력한 문자열을 포함하는 이름을 가지는 버킷을 검색할 수 있습니다.

| 항목 | 필수 | 설명 | 제한 |
| --- | --- | --- | --- |
| 검색어 | O | 검색할 버킷의 이름 | 영문 소문자와 숫자, 특수문자('`.`', '`-`')를 사용 가능합니다. |

### 버킷 상세 조회

버킷 목록에서 버킷 이름을 제외한 나머지 영역을 클릭하여 버킷의 상세 정보를 조회할 수 있습니다.

| 항목 | 설명 |
| --- | --- |
| 버킷 이름 | 버킷의 이름 |
| 설명 | 버킷 생성 시 등록한 설명 |
| 객체 개수 | 버킷에 저장된 객체의 수 |
| 크기 | 버킷에 저장된 객체의 총 크기 |
| 생성 시간 | 버킷의 생성 시각 |
| 최근 업데이트 시간 | 버킷이 최근 업데이트된 시간 |

## 버킷 관리하기

### 버킷 생성

버킷 목록 상단의 **버킷 생성**을 클릭해 버킷을 생성할 수 있습니다. 버킷 생성 시 필요한 입력 값은 아래 표와 같습니다.

!!! tip "알아두기"
    Data Lake Storage 서비스의 버킷 이름은 리전 유니크합니다.

| 항목 | 필수 | 설명 | 제한 |
| --- | --- | --- | --- |
| 이름 | O | 버킷의 이름 | 3 \~ 63자의 길이를 가져야 합니다.<br>영문 소문자와 숫자, 특수문자('`.`', '`-`')를 사용 가능합니다.<br>이름의 시작과 끝에는 영문 소문자 또는 숫자를 사용해야 합니다.<br>연속된 `.`(`..`), IPv4 패턴, 일부 접두사 및 접미사는 사용할 수 없습니다.<br>사용 불가능한 접두사: `xn--`, `sthree-`, `amzn-s3-demo-`<br>사용 불가능한 접미사: `-s3alias`, `--ol-s3`, `.mrap`, `--x-s3`, `--table-s3` |
| 설명 | X | 버킷의 설명 | 100자 이내의 길이를 가져야 합니다. |

### 버킷 삭제

버킷 목록의 체크박스를 사용하여 버킷을 선택한 후, 버킷 목록 상단의 **삭제**를 클릭하여 버킷을 삭제할 수 있습니다.

!!! danger "주의"
    삭제할 대상은 객체가 모두 제거된 빈 버킷이어야 합니다. 버킷이 비어 있지 않은 경우, 객체 제거 API나 버킷 비우기 기능을 사용하여 버킷을 먼저 비워야 합니다.

버킷 제거는 되돌릴 수 없습니다.

### 버킷 비우기

버킷 목록의 체크박스를 사용하여 버킷을 선택한 후, 버킷 목록 상단의 **버킷 비우기**를 클릭하여 버킷 내 모든 객체 제거를 요청할 수 있습니다.

| 항목 | 필수 | 설명 | 제한 |
| --- | --- | --- | --- |
| 이름 | O | 비울 버킷의 이름 |  |

!!! danger "주의"
    객체 수에 따라 시간이 소요될 수 있습니다.

버킷 비우기 작업은 도중에 취소하거나 되돌릴 수 없습니다.

## 자격 증명 관리하기

### 자격 증명 조회

버킷 목록 조회 페이지 상단의 **S3 API 자격 증명**을 클릭하면 자격 증명 관리 팝업을 표시합니다. 팝업 내에서 발급된 자격 증명 목록을 확인할 수 있습니다.

| 항목 | 설명 |
| --- | --- |
| Access Key | 자격 증명의 Access Key |
| 설명 | 자격 증명 발급 시 등록한 설명 |
| 생성 일시 | 자격 증명의 생성 일시 |
| 최근 사용 일시 | 자격 증명의 최근 사용 일시 |
| 관리 | 수정 및 삭제 기능 |

### 자격 증명 발급

S3 API 자격 증명 팝업 상단의 **S3 API 자격 증명 등록**을 클릭하여 새로운 자격 증명을 발급할 수 있습니다. 자격 증명 발급 시 사용 목적 등의 설명을 추가할 수 있습니다.

| 항목 | 필수 | 설명 | 제한 |
| --- | --- | --- | --- |
| 설명 | X | 자격 증명의 용도 등 설명 | 100자 이내 |

!!! tip "알아두기"
    사용자 계정별로 한 프로젝트당 S3 API 자격 증명을 최대 3개 발급할 수 있습니다.

!!! danger "주의"
    API를 사용할 때 사용자 인증을 위해 자격 증명을 사용합니다. 유출 시 타인이 저장소의 기능과 객체에 접근할 수 있습니다. 반드시 안전한 위치에 저장하고 다른 사람과 공유하지 마세요.
    자격 증명 발급 직후 Access Key와 Secret Key가 표시됩니다. Secret Key는 발급 직후 1회만 확인 가능합니다.

### 자격 증명 수정

S3 API 자격 증명 목록 관리 영역의 **수정**을 클릭하여 자격 증명의 설명을 수정할 수 있습니다.

| 항목 | 필수 | 설명 | 제한 |
| --- | --- | --- | --- |
| 설명 | X | 자격 증명의 용도 등 설명 | 100자 이내 |

### 자격 증명 삭제

S3 API 자격 증명 목록 관리 영역의 **삭제**를 클릭하여 자격 증명을 삭제할 수 있습니다. 이때 확인 팝업이 표시됩니다.

!!! danger "주의"
    삭제한 자격 증명으로는 API 인증을 할 수 없습니다. 또한 삭제한 자격 증명은 복구할 수 없습니다.

## 객체 조회하기

### 객체 목록 조회

버킷 목록에서 버킷 이름을 클릭하여 버킷 내의 객체 목록을 조회할 수 있습니다.

| 항목 | 설명 |
| --- | --- |
| 이름 | 객체의 이름 |
| 크기 | 객체의 크기 |
| 최근 업데이트 시간 | 객체를 업로드한 시각 |

### 객체 상세 조회

객체 목록에서 파일인 객체를 클릭하여 객체의 상세 정보를 조회할 수 있습니다.

| 항목 | 설명 |
| --- | --- |
| 객체 이름 | 객체의 이름 |
| 크기 | 객체의 크기 |
| ETag | 객체의 버전 정보 |
| 최근 업데이트 시간 | 객체를 업로드한 시각 |
| 스토리지 클래스 | 객체의 스토리지 클래스 |
